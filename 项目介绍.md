# 🎬 通俗动画剧本工坊 — 项目介绍

> 一个用数据驱动创作的AI动画短片方法论系统

---

## 一句话说明

**拆解98部全球经典动画短片，提炼15条创作规律，用AI工具在2分钟内讲一个北美观众会转发的好故事。**

---

## 🎯 目标

创作**时长2分钟以下、适合北美口味、动画形式、面向流媒体平台**（YouTube Shorts / TikTok / Instagram Reels）传播的短片内容。

具体来说：
- 建立一套**可复用的创作方法论**（从选题到成片）
- 产出**3-5个可执行的动画剧本**
- 验证**AI动画制作工作流**（从剧本到成片的全链路）

---

## ⚙️ 原理

### 核心思路：先拆后建

我们没有从零开始"凭感觉"写剧本，而是采用了**逆向工程**的方法：

```
拆解经典作品 → 提炼共性规律 → 构建创作公式 → 用公式生成新故事
```

### 具体做了什么

**第一步：大规模拆片**

跨越13个维度，拆解了**98部**全球动画短片/片段：

| 维度 | 涵盖内容 |
|------|---------|
| 流媒体喜剧 | Slapstick / 反讽 / 荒诞 / 黑色幽默 各Top5 |
| 流媒体情感 | 温暖 / 孤独 / 怀旧 / AI新锐 各Top5 |
| 头部账号 | Alan Becker / Cas van de Pol / worthikids 等6个频道 |
| 传统电影节 | 皮克斯 / 奥斯卡 / 昂西 获奖短片15+ |
| 日本/亚洲美学 | 今敏 / 新海诚 / 宫崎骏 / 水墨动画 12部 |
| 游戏PV/CG | Riot / 暴雪 / 米哈游 / 独立游戏 9部 |
| 音乐MV | Gorillaz / A-ha / Daft Punk / YOASOBI 8部 |
| 商业+社媒+AI | 品牌短片 / TikTok原生 / AI动画 6部 |

**第二步：提炼规律**

从98部作品中归纳出**15条创作规律**，覆盖叙事、受众、情感、视觉、传播、制作6个层面：

| 类别 | 核心规律（举例） |
|------|----------------|
| 叙事 | 2min只容一个概念；七种结构模板；结尾决定一切 |
| 受众 | 三个圈层（全年龄→Gen Z→影迷）；幽默类型排名 |
| 视觉 | 风格化 > 写实；AI不完美 = 风格 |
| 音乐 | 音乐 = 叙事引擎，承担50%+叙事功能 |
| 差异化 | 东方美学 = 蓝海；文化混搭 = 传播乘法 |
| 制作 | 一人工作室时代；AIGC约束 = 创意约束 |

**第三步：构建公式**

将规律浓缩为一个**可操作的创作公式（V3.0）**：

```
2分钟以下北美向AI动画 = 
  一句话概念 + 非人类主角
  + 7种结构中选1种 + 零对白或音乐驱动
  + 风格化视觉（辨识度 > 精度）
  + 1个视觉记忆点 + 文化混搭加分项
  + 强结尾 + AIGC友好设计
  + ≤2个主角，≤10个镜头
  + 配乐/节奏设计 ≥ 画面本身
```

**第四步：用公式生成故事**

基于公式，搭建了**故事生成器**——选几个标签，AI自动匹配最相关的作品、规律和创意，一键生成故事梗概，再进入共创对话逐轮迭代。

| 📺 头部账号 | 6个标杆频道的运营分析 |

| 💡 批量创意 | 20个公式拼配的故事种子 |

### 3. 故事生成器：从标签到剧本

不是空白页面对着光标发呆——是：
- **选标签**：6个维度（主角类型 / 动画风格 / 情感基调 / 叙事结构 / 时长 / 题材），选2-6个
- **智能匹配**：自动从98部作品、15条规律、20个创意中匹配最相关的参考
- **AI生成**：一次产出3-5个200字故事梗概，每个用不同的规律和参考组合
- **共创迭代**：选中喜欢的，进入Chat模式和AI编剧导师逐轮打磨

### 4. 独立版本，一个文件走天下

`dashboard-standalone.html` —— 236KB的单个HTML文件，双击就能打开。无需安装任何东西，无需启动服务器。发给朋友就能看到全部调研成果。

### 5. 跨文化视角

调研覆盖了皮克斯/迪士尼（北美主流）、日本anime（今敏/新海诚/宫崎骏）、中国水墨（山水情/秋实）、游戏CG（Riot/暴雪/米哈游）、独立动画（worthikids/Cyriak）——不是只盯着一个文化圈。

## 🛠️ 作用

这个系统解决的核心问题是：**如何让一个人（或小团队）用AI工具，高效地创作出能在北美流媒体平台传播的动画短片。**

具体来说：

| 使用场景 | 怎么用 |
|---------|--------|
| **我要写一个新剧本** | 打开故事生成器 → 选标签 → AI生成梗概 → 共创迭代 → 用检查清单验证 |
| **我想看看什么故事受欢迎** | 浏览作品库 → 按类型/风格/情感筛选 → 看拆片分析 |
| **我不确定这个想法好不好** | 对照12项检查清单逐条打分 → 低于8项通过就回炉重造 |
| **我想学习大师怎么讲故事** | 看规律总结 → 看叙事结构模板 → 看对应的经典作品分析 |
| **我想找创作灵感** | 浏览20个批量创意 → 或在生成器里随机组合标签 |
| **我想给别人展示这个项目** | 发送 `dashboard-standalone.html` 一个文件搞定 |

## 📁 技术架构（简要）

```
data/*.js (7个数据文件)     ← 唯一的数据源（Single Source of Truth）
       ↓
dashboard.html              ← 渲染层（10个板块的仪表盘）
       ↓
server.py                   ← 本地服务器 + OpenRouter API代理（AI生成功能）

dashboard-standalone.html   ← 独立版本（JS内联，双击可用）
```

- **数据与视图分离**：所有内容存在7个JS文件中，HTML只负责渲染
- **AI接口**：通过OpenRouter统一调用Claude / GPT / Gemini / DeepSeek等多种模型
- **零依赖**：纯前端HTML+JS+CSS，无框架、无构建工具、无数据库
