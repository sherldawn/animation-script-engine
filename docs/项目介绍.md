# 🎬 2min动画短片 — 智能创作系统 · 项目介绍

> 一个数据驱动的动画短片创作系统：拆解98部经典作品，提炼15条规律，用AI帮你写剧本。

---

## 一句话说明

**这个系统用拆解98部全球经典动画短片的方式，提炼出15条创作规律，再结合AI编剧导师，帮用户从零写出一个能在北美流媒体平台传播的2分钟动画剧本。**

---

## 🎯 目标

创作**时长2分钟以下、适合北美口味、动画形式、面向流媒体平台**（YouTube Shorts / TikTok / Instagram Reels）传播的短片内容。

具体来说：
- 建立一套**可复用的创作方法论**（从选题到成片的完整流程）
- 搭建一个**可交互的创作工具**（选标签 → AI生成 → 共创对话 → 导出剧本）
- 产出**3-5个可执行的动画剧本**
- 验证**AI动画制作工作流**（从剧本到成片的全链路）

---

## ⚙️ 原理：先拆后建

这个系统没有凭感觉写剧本，而是采用了**逆向工程**的方法：

```
拆解经典作品 → 提炼共性规律 → 构建创作公式 → 用公式+AI生成新故事 → 和AI导师共创迭代
```

### 第一步：大规模拆片

跨越13个维度，拆解了**98部**全球动画短片/片段：

| 维度 | 涵盖内容 |
|------|---------|
| 流媒体喜剧 | Slapstick / 反讽 / 荒诞 / 黑色幽默 各Top5 |
| 流媒体情感 | 温暖 / 孤独 / 怀旧 / AI新锐 各Top5 |
| 头部账号 | Alan Becker / Cas van de Pol / worthikids 等6个频道 |
| 传统电影节 | 皮克斯 / 奥斯卡 / 昂西 获奖短片15+ |
| 日本/亚洲美学 | 今敏 / 新海诚 / 宫崎骏 / 水墨动画 12部 |
| 游戏PV/CG | Riot / 暴雪 / 米哈游 / 独立游戏 9部 |
| 音乐MV | Gorillaz / A-ha / Daft Punk / YOASOBI 8部 |
| 商业+社媒+AI | 品牌短片 / TikTok原生 / AI动画 6部 |

每部作品都有：完整剧情描述、亮点分析、6维标签标注、关键词索引、外部链接。

### 第二步：提炼规律

从98部作品中归纳出**15条创作规律**，覆盖叙事、受众、情感、视觉、传播、制作6个层面：

| 类别 | 核心规律（举例） |
|------|----------------|
| 叙事 | 2min只容一个概念；七种结构模板；结尾决定一切 |
| 受众 | 三个圈层（全年龄→Gen Z→影迷）；幽默类型排名 |
| 视觉 | 风格化 > 写实；AI不完美 = 风格 |
| 音乐 | 音乐 = 叙事引擎，承担50%+叙事功能 |
| 差异化 | 东方美学 = 蓝海；文化混搭 = 传播乘法 |
| 制作 | 一人工作室时代；AIGC约束 = 创意约束 |

### 第三步：构建公式

将规律浓缩为一个**可操作的创作公式（V3.0）**：

```
2分钟以下北美向AI动画 = 
  一句话概念 + 非人类主角
  + 7种结构中选1种 + 零对白或音乐驱动
  + 风格化视觉（辨识度 > 精度）
  + 1个视觉记忆点 + 文化混搭加分项
  + 强结尾 + AIGC友好设计
  + ≤2个主角，≤10个镜头
  + 配乐/节奏设计 ≥ 画面本身
```

### 第四步：用公式+AI生成故事，再共创迭代

基于公式和全部数据，搭建了**故事生成器 + AI编剧共创系统**：

1. 用户选几个标签描述想要的方向
2. 系统从98部作品和15条规律中智能匹配，组装完整Prompt
3. AI一次生成3-5个不同方向的故事梗概
4. 用户选中喜欢的方向，进入Chat模式和AI编剧导师逐轮打磨
5. AI导师每次修改都会引用具体规律编号给出理由，并更新12项检查清单评分
6. 用户满意后导出完整剧本（Markdown格式）

## ✨ 亮点

### 1. 不是Prompt生成器，是完整的创作系统

很多工具只做到"帮你生成一段文字"。这个系统做的是**从选题到成片的全流程**：

- **智能Roll点引擎**：6维度×66标签的组合系统，每次生成方向都不同
- **规律轮盘**：每个生成的故事随机使用不同的创作规律作为驱动核心
- **DNA混搭**：从98部作品中随机抽取2部做基因重组，产生意想不到的组合
- **AI编剧共创对话**：选中方向后进入Chat模式，和AI导师逐轮打磨，不是生成一次就扔给用户
- **12项检查清单**：每轮迭代都有量化评分，用户清楚知道剧本哪里好、哪里需要改
- **一键导出**：最终剧本+全部对话记录导出为Markdown文件

### 2. 数据库级的调研深度

不是随便看几个视频就开写——是98部作品 × 13个维度 × 每部作品逐秒拆解。所有数据结构化存储在7个JS文件中，随时可查、可比较、可引用。

| 📺 头部账号 | 6个标杆频道 × 3部代表作 |

调研覆盖了皮克斯/迪士尼（北美主流）、日本anime（今敏/新海诚/宫崎骏）、中国水墨（山水情/秋实）、游戏CG（Riot/暴雪/米哈游）、独立动画（worthikids/Cyriak）——不是只盯着一个文化圈。

## 📁 技术架构

```
data/*.js (7个数据文件)         ← 所有数据的唯一来源（Single Source of Truth）
       ↓
dashboard.html                  ← 开发版（加载外部JS文件，需HTTP服务器）
index.html                      ← 独立版（所有JS内联，双击可用，242KB）
       ↓
OpenRouter API                  ← AI生成（前端直连OpenRouter，无需后端）
```

- **数据与视图分离**：所有内容存在7个JS文件中，HTML只负责渲染和交互
- **前端直连**：AI生成功能直接调用OpenRouter API，不需要后端服务器
- **支持多模型**：通过OpenRouter统一调用Claude / Gemini / DeepSeek等模型
- **零依赖**：纯前端HTML + JS + CSS，没有框架、没有构建工具、没有数据库
