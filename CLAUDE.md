## 🔄 工作模式（必须遵守）

### 规则1：先方案，后执行（Plan-then-Act）

**每一次用户发起新对话或新任务时，Claude 必须：**

1. **先在聊天框中给出方案**（明确列出要做什么、怎么做、预期产出）
2. **等待用户确认**（"确认"/"OK"/"可以"/"改一下XXX"）
3. **用户确认后才开始执行**

❌ 禁止：直接开始写文件、跑命令，不给方案就动手
✅ 正确：先说"我计划做以下几件事：1... 2... 3... 你确认后我开始执行"

### 规则2：使用 Git 管理所有产出

- 项目目录必须初始化为 git 仓库
- 每完成一个有意义的里程碑，都要 `git add` + `git commit`
- Commit message 使用中文，格式：`[板块] 具体内容`

### 规则3：永不停止，持续迭代（Never Stop）

- 完成一个阶段后，立即评估当前进度，找到下一个需要推进的板块
- **除非用户明确说"停下来"或"暂停"，否则永远继续工作**

### 规则4：进度追踪

- 使用 `task_progress` 参数跟踪进度
- 每次工具调用都更新 checklist

## 📁 项目文件结构

```
通俗动画剧本1min/
├── CLAUDE.md                      ← 本文件（系统指令）
├── dashboard.html                 ← 调研与创作仪表盘（纯展示层）
│
├── data/                          ← 🔑 唯一数据源（Single Source of Truth）
│   ├── tags-registry.js           ← 标签体系定义
│   ├── works.js                   ← 作品库（51部）
│   ├── config.js                  ← 结构 + 方向 + 创意工坊
│   ├── batch-ideas.js             ← 批量创意（20个）
│   ├── research.js                ← 调研数据（概览/案例/账号/电影节）
│   └── rules.js                   ← 规律 + V2.0公式 + 检查清单
│
├── 剧本/
│   ├── 001-火锅毛肚大战.md         ← V1→V4完整迭代记录
│   └── 批量创意20.md               ← 20个批量创意的MD源文件
│
├── 方法论/                         ← 后续放V2.0方法论
│
├── 存档/                           ← 调研MD源文件（已迁入data/*.js）
│   ├── AI短片案例分析.md
│   ├── 调研方案：2min以下北美动画短片.md
│   ├── 传统电影节线-按分类拆片.md
│   ├── 规律总结.md
│   ├── 流媒体线-拆片-情感类.md
│   ├── 流媒体线-拆片-喜剧类.md
│   ├── 流媒体线-头部账号Top3作品.md
│   └── 维度9-中国文化×北美市场.md
│
└── .git/
```

### 架构原则

- **data/*.js = 唯一数据源**，所有内容修改都在这里进行
- **dashboard.html = 纯展示层**，不含任何数据，从 data/ 渲染
- **存档/ = 历史参考**，不是 single source of truth
- **剧本/ = 创意源文件**，详细版在MD，精简版在 data/*.js
- 新增调研/维度/创意 → 直接编辑对应的 data/*.js 文件
- 需要 HTTP 服务器加载（`python3 -m http.server 8888`）
