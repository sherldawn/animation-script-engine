## 🎯 项目目标

**创作时长2分钟以下、适合北美口味、动画形式、流媒体平台传播的短片内容。**

具体目标拆解：
1. 完成系统性市场调研（平台规则、受众画像、竞品分析、叙事结构、工具链、商业模式）
2. 基于调研产出《创作方法论V2.0》
3. 产出3-5个可执行的1-2分钟动画剧本
4. 建立可复制的AI动画制作工作流

## 📐 评测标准

每一轮工作产出，都用以下10个问题来检验完成度：

| # | 问题 | 通过标准 |
|---|------|---------|
| 1 | 作品应该发在哪个平台？ | 能说出平台+理由+发布策略 |
| 2 | 北美观众最吃哪种幽默？ | 能列出3种并给出案例 |
| 3 | 目标受众是谁？ | 能描述年龄、习惯、偏好 |
| 4 | 2min以下有哪些可用的叙事结构？ | 能列出5种并各给案例 |
| 5 | 竞争对手是谁？ | 能说出10个频道/创作者 |
| 6 | 他们做得好的地方是什么？ | 每个能说出1-2个亮点 |
| 7 | 用什么工具制作？ | 有完整的工具链方案 |
| 8 | 一条片子成本多少、耗时多久？ | 有具体数字 |
| 9 | 怎么赚钱？ | 有至少2条变现路径 |
| 10 | 第一条片子应该是什么？ | 有一个明确的剧本方向 |

## 🔄 工作模式（必须遵守）

### 规则1：先方案，后执行（Plan-then-Act）

**每一次用户发起新对话或新任务时，Claude 必须：**

1. **先在聊天框中给出方案**（明确列出要做什么、怎么做、预期产出）
2. **等待用户确认**（"确认"/"OK"/"可以"/"改一下XXX"）
3. **用户确认后才开始执行**

❌ 禁止：直接开始写文件、跑命令，不给方案就动手
✅ 正确：先说"我计划做以下几件事：1... 2... 3... 你确认后我开始执行"

### 规则2：使用 Git 管理所有产出

- 项目目录必须初始化为 git 仓库
- 每完成一个有意义的里程碑，都要 `git add` + `git commit`
- Commit message 使用中文，格式：`[板块] 具体内容`，例如：
  - `[调研] 完成平台规则速查表`
  - `[拆片] 新增15个标杆案例深度分析`
  - `[剧本] 第一版动物喜剧剧本`
  - `[方法论] 更新创作公式V2.0`

### 规则3：永不停止，持续迭代（Never Stop）

- 完成一个阶段后，立即评估当前进度，对照评测标准，找到下一个需要推进的板块
- 如果所有板块都完成了，进入剧本创作阶段，写完一个剧本就写下一个
- 如果剧本写完了，回头用评测标准重新审视，迭代优化
- **除非用户明确说"停下来"或"暂停"，否则永远继续工作**

迭代循环：
```
调研 → 分析 → 方法论 → 剧本 → 自评 → 优化 → 更多剧本 → ...
```

### 规则4：进度追踪

- 使用 `task_progress` 参数跟踪进度
- 每次工具调用都更新 checklist
- 让用户随时能看到"做到哪了"
